Intent

The operating model for AI-augmented teams

When AI collapses implementation from weeks to hours, the bottleneck moves upstream — from delivery to discovery, specification, and observation. Intent replaces ceremony-driven coordination with a continuous signal-driven loop that scales from a single practitioner to teams of teams working alongside hundreds of agents.

One Loop, Any Scale

The same four-phase loop operates at every level of scale. What changes isn't the model — it's the volume of signals and the autonomy level of the agents processing them.

Notice

Capture signals from every surface — conversations, code reviews, agent traces, OTel spans, Slack threads. A signal is anything worth paying attention to. One person might capture five a day. A fleet of agents might surface five thousand.

Spec

Shape signals into agent-ready specifications. Specs are contracts, not stories — verifiable assertions that an AI agent can execute against. Clear specs mean agents work autonomously; vague specs mean expensive human intervention.

Execute

Agents implement against specs at the autonomy level earned by the signal's trust score. Low trust means human approval. High trust means full autonomy with circuit breakers. The system learns which work it can handle alone.

Observe

Every action emits structured events. Observe what happened, compare it to what was specified, and feed the delta back as new signals. The loop closes. The system gets smarter. The next cycle starts with better information.

Signals Are the Architecture

Most AI tooling focuses on the execute phase — making agents that can write code or draft emails. Intent focuses upstream, on the signal layer that decides what gets executed and how much autonomy the executor gets. Signals are the primitive. Everything else follows.

Trust-Scored

Every signal gets a trust score computed from clarity, blast radius, reversibility, testability, and precedent. Trust determines autonomy level — from L0 (human drives) to L4 (full agent autonomy, circuit breakers only).

Amplification-Weighted

Signals gain weight through repeated reference. The same friction point surfacing across conversations, commits, and agent traces tells the system this matters more than its initial score suggested. Frequency is information.

Self-Clustering

Signals that get referenced together reveal emergent problem structure. The system discovers its own architecture — not through top-down design, but through the pattern of what keeps showing up together.

Pipeline-Processed

An enrichment pipeline works every signal as far as it can go: dedup, contextualize, score, classify, route. Humans intervene only when something is ambiguous or unsafe. The system never dead-ends — it generates disambiguation signals instead.

Scale by Design

Intent is architected to grow. The same abstractions that work for one person noticing friction work for an organization processing thousands of signals across dozens of teams and hundreds of agents.

Where Intent Is Now

Intent is methodology-first — validate with practitioners before building software. Everything is file-native, git-tracked, and OTel-compatible. No lock-in, no proprietary formats.

Notice — Operational

MCP server, CLI suite, GitHub Action, signal template. 15 founding signals captured, trust-scored, and clustered. Signal amplification spec drafted.

Spec — Conceptual

Templates for intents, specs, contracts. CLI tools for proposing, approving, and tracking. Seven-level work ontology defined.

Execute — Defined

Event schema designed with 15 OTel-compatible events. Agent handoff protocol documented. No integration yet — this is next.

Observe — Schema-Ready

Event types defined, signal dashboard live, trust visualization operational. Full dashboard and learning layer planned.
