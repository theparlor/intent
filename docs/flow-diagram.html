Flow Diagram

Five concrete paths through the system with personas (Architect, PM, Design/QA, Agent) and event triggers at every transition.

Path 1: Architect Captures Signal

Signal → Intent → Approval

Actor: Architect
Trigger: Cowork session or observation from agent traces
Event: signal.created
Next: PM reviews signal and links to intent

Path 2: PM Writes Intent

Intent → Spec Assignment

Actor: PM
Trigger: Signal reaches confidence threshold
Event: intent.proposed, intent.approved
Next: Designer/Architect writes spec with contracts

Path 3: Design/QA Writes Spec

Spec → Contract → Ready for Execution

Actor: Designer/QA
Trigger: Intent approved
Events: spec.written, spec.staged
Next: Agent picks up staged spec

Path 4: Agent Executes Contract

Contract → Execution → Observation

Actor: Agent
Trigger: Spec staged and dependencies resolved
Events: contract.started, contract.assertion.passed/failed, contract.completed
Next: Observe layer reads traces, writes findings

Path 5: Observe Closes Loop

Traces → Observations → New Signals

Actor: Agent (observe phase)
Trigger: Contract completed and all traces aggregated
Events: trace.completed, observation.written, signal.created
Next: Loop continues — observations feed back into notice phase
