Event Catalog

15 events, 6 emission mechanisms, OTel-compatible schema. What gets captured and how, from GitHub Actions to agent self-emission.

All events follow OpenTelemetry conventions and include timestamp, source, trace ID, and span context for end-to-end observability.

signal.created

Emitted when a new signal is captured from work, research, or conversation.

intent.proposed

Emitted when an intent is created, linking signals to what we want to change.

intent.approved

Emitted when an intent receives approval to proceed to spec.

spec.written

Emitted when a spec is created with narrative, acceptance criteria, and contract assertions.

spec.staged

Emitted when a spec moves from draft to ready-for-execution.

contract.started

Emitted when an agent begins executing a contract.

contract.assertion.passed

Emitted when a contract assertion passes.

contract.assertion.failed

Emitted when a contract assertion fails.

contract.completed

Emitted when all assertions pass and contract is fulfilled.

capability.released

Emitted when a capability (reusable building block) is added to the system.

feature.released

Emitted when a feature ships to production.

decision.recorded

Emitted when a significant decision is made and documented.

trace.completed

Emitted when an agent completes execution and all traces are aggregated.

observation.written

Emitted when the observe layer reads traces and writes findings.

system.health

Periodic emission of system health metrics: velocity, quality, observability coverage.
